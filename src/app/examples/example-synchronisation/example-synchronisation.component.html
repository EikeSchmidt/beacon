Since it looks awkward when all beacons created at the same time blink at the same time, every beacon starts delayed by a random dozen of milliseconds.
If you give beacons the same synchronisation-id (must be a small integer), they get the same delay and will blink synchronously.

That effect only takes place if the beacon get created at the same time. That is why you have to recreate the beacon in the example with *ngIf.

You can make all simultaneously created beacons synchronous with the beacon service by beacon.setDefaultSynchronisation(true).
